Project4. APG WebShop

1.General task specification:
In this project, we implement and deploy a working webshop in Java as a EJB-based Web-application. It is allowed for customers to register, logon, logout, browse or search the information of gnomes (including name, price, quantity and description), put units into shopping cart, check the state of cart, and pay the bill by credit-card. Besides, administrator can log on to system, manage the customers, manage the gnomes (browse, add and remove).

2.Information about HW platform:
A computer with XP or Vista operating system. (minimum 1G RAM)

3.List of Java software and programming technologies:
  • Java software: Java SE, Java EE(include JSF Library), MySQL, Netbeans, Apache Tomcat
  • Programming technology: JSP, JSF, JDBC

4.Detailed description of application:
  • Use Case Diagram:



  • Structure: Our application adopts the three-tier architecture: customer(browser) in the first tier, uses the services provided by the webserver in the 2nd tier, which communicates with database and instructs the manipulation of data.

    •

 

  • Functionality:
    • Customer can register itself, and logon, logout. (The password of customer is encoded with MD5 algorithm to ensure the security of customer's information)

         

      • Customer is able to search and browse the information of gnomes, like inventory or details. Of course the gnomes available can be put into shopping cart by customers.

        • Check the state of shopping cart and make order by paying bills online.

      • Administrator also be able to log on to the system, and manage the gnomes, like add and remove gnome.

        Check all the orders that customers have made.

      • Manage the customers, like browse list of all members, and block or unblock some a customer.

  • Communication protocol: HTTP protocol.

  • Interactions: The interactions between customers and webserver are processing through inputting informations and pressing button by customers and through showing corresponding result in pages by webserver.



5.Instructions:
Installation instruction
      The following are the steps of deploying the project (The Tomcat server and MySQL server must be in a same machine. Notice: The root password of MySQL should be "123456" in default, it it is not, you have to change the source code. Modify the dbPassword field member in DatabaseUtil class in the source code.)
    1. Create a database named "webshop" in MySQL server
    2. Start a console and locates to the directory of webshop.sql file
    3. Type the following command in console to import the data into MySQL
      4. mysql -u root -p webshop < webshop.sql
    4. Copy GnomeShop.war into TOMCAT_HOME\webapps\ (TOMCAT_HOME is the Tomcat installation directory)
    5. Start Tomcat server
Running instruction:

      Please follow the instructions below to run the application.

      Start Internet Explorer and enter the following url: http://ServerAddress:8080/GnomeShop/ (The ServerAddress is the ip address which the web server locates)

      The following are the steps of buying gnomes on the web
    1. First login into the system, click the "Login" hyperlink on the top panel to login. here we can use username "guest" and password "guest"
    2. Click "Browse All Gnomes" on the left panel and you will see the list of all gnomes
    3. Click the "Details" to check the detail information of each kind of gnomes
    4. Enter the number of gnomes and click "Add into My Cart" to add the gnomes into shopping cart. You can repeat the step 3 to 5 to buy several kinds of gnomes
    5. Click "ShoppingCart" link in the up panel to check the shopping cart
    6. Click "Checkout" enters the checkout page, here you have to enter the order information, e.g. contact name, delivery address, etc.
    7. Click "Submit" button to submit the order and wait for gnomes ^_^
      The following are the steps of registering a member
    1. Click the "Register" hyperlink on the top panel to enter registering page.
    2. Enter the required information into the input text boxes
    3. Click "Submit" button and you will see the registration results. If the registration failed, which means the user name is already in the database, you may repeat step1 and 2 and choose another username
      The following are the steps for administrators to remove a product or add a new kind of product into repository
    1. First login into the system, click the "Login" hyperlink on the top panel to login. here we can use username "admin" and password "admin", if you are already logged in, click to logout first
    2. Click "Manage" hyperlink on the top panel, you may see all the gnomes in the repository. You can click "Remove" to remove a certain kind of gnome
    3. If you want to add a new kind of products into the repository, you may click "Add Gnome" link to enter the adding page
    4. In the adding page, enter the required information of new gnomes and click "Add" button. If the adding succeeds then you will see the new gnomes are added in the gnome list
      The following are the steps for administrators to block members
    1. First login into the system, click the "Login" hyperlink on the top panel to login. here we can use username "admin" and password "admin", if you are already logged in, click to logout first
    2. Click "Admin" hyperlink on the top panel, you may see all the members registered. You can check their registration information by clicking the "Details" hyperlink
    3. In the detail page, you can block/unblock a member. If you block the member, he/she cannot log into the system with his/her username and password anymore.
      The following are the steps for administrators to check all the orders
    1. First login into the system, click the "Login" hyperlink on the top panel to login. here we can use username "admin" and password "admin", if you are already logged in, click to logout first
    2. Click "Orders" hyperlink on the top panel, you may see all the orders which customer creates. You can check the order detail by clicking "Details" hyperlink.
